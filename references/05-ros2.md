# ROS 2 集成（6.0.1）

## Bridge 架构与支持版本

- 官方推荐 **ROS 2 Humble 与 Jazzy**；实验性支持 Ubuntu 22.04/24.04 上任意原生安装的 ROS 2 发行版。无系统级 ROS 时可用**内置 ROS 2 库**（适合容器化：仿真与机器人软件分离运行）。
- 集成方式两种：① **ROS 2 OmniGraph（OG）节点**（Action Graph 中连线）；② **内置 Python 3 环境 + rclpy** 编写自定义节点/服务/action。
- 典型流程：URDF 导入（转 USD）→ 搭建环境（SimReady 资产、Warehouse Creator）→ 加传感器（RGB/RGBD 相机、RTX Lidar/Radar、IMU、接触传感器）→ 配置 Bridge → 仿真 → 对接 ROS 2 栈。

来源: https://docs.isaacsim.omniverse.nvidia.com/6.0.1/ros2_tutorials/ros2_reference_architecture.html、ros2_landing_page.html

## 6.0 OmniGraph 节点变化（迁移重点）

从 5.1 及更早版本升级的 Action Graph 必须改造：两个发布节点**不再内部解析 prim**，改为接收前置节点的预计算数据（直接 prim 输入已弃用）：

- **ROS2 Publish Transform Tree** → 前置新增 **Isaac Compute Transform Tree**（`isaacsim.core.nodes`，设 `targetPrims`/可选 `parentPrim`），连接 `execOut→execIn`、`parentFrames/childFrames/translations/orientations` 同名对接。
- **ROS2 Publish Joint State** → 前置新增 **Isaac Read Joint State**（`isaacsim.sensors.physics.nodes`，`prim` 设为 articulation root），对接 `jointNames/jointPositions/jointVelocities/jointEfforts/jointDofTypes/stageMetersPerUnit/sensorTime`。
- 另：**ROS2 Camera Helper / Camera Info Helper 的 `frameSkipCount` 输入弃用**，改在相机 prim 上设 `omni:sensor:tickRate`（多 tick 渲染）。

来源: migration_guides/isaac_sim_6_0/ros2_omnigraph_migration.html

## 发布订阅与 QoS

- QoS 通过节点的 `qosProfile` JSON 字符串配置：`history(keepLast)/depth(整数)/reliability(reliable|bestEffort)/durability(volatile|transientLocal)/deadline/lifespan/liveliness/leaseDuration`；或用 **ROS2 QoS Profile** 节点自动生成（含 Sensor Data 等预设，改字段前先把 createProfile 设为 Custom）。RViz 看传感器数据建议 Best Effort。
- 通用收发：Generic Publisher/Subscriber、Generic Server/Client、Prim Attribute Service 教程可对任意消息类型收发与改 prim 属性。

来源: ros2_tutorials/tutorial_ros2_qos.html

## 时钟

- 发布仿真时间：**On Playback Tick + ROS2 Context + Isaac Read Simulation Time + ROS2 Publish Clock** → `/clock`；换 **Isaac Read System Time** 可发墙钟。外部节点 `ros2 param set /node use_sim_time true`。菜单 Tools > Robotics > ROS 2 OmniGraphs > Clock 一键生成。另有 RTF 发布教程（tutorial_ros2_rtf.html）。

来源: ros2_tutorials/tutorial_ros2_clock.html

## 相机与 Lidar/Radar 发布（含压缩图像）

- 相机：**Isaac Create Render Product + ROS 2 Camera Helper / Camera Info Helper**，type 选 rgb/depth/点云/bbox/语义等，发布 `rgb`、`depth`、`camera_info`。菜单可一键生成。camera_publishing 教程给三种方式：Replicator writer（`ROS2PublishImage/ROS2PublishCameraInfo/ROS2PublishPointCloud`）、OG 节点、helper 函数；多 tick 渲染下由相机 `tick_rate` 决定发布频率，无需分频节点（新 API：`isaacsim.sensors.experimental.rtx` 的 CameraSensor/RtxCamera）。
- **压缩图像（6.0 新增）**：Camera Helper type 设 `rgb_h264`，发 `sensor_msgs/CompressedImage`（format=h264，每帧为完整 IDR 帧）到 `image_raw/compressed`；用 `isaac_compressed_image_decoder` 包解码到 `/image_decoded`。
- RTX Lidar：**Isaac Run One Simulation Frame + Isaac Create Render Product + ROS 2 RTX Lidar Helper**；2D 用 type=laser_scan→`/scan`，3D 用 point_cloud→`/point_cloud`（勾选 Publish Full Scan）；配置如 `Example_Rotary(_2D)`。示例：`./python.sh standalone_examples/api/isaacsim.ros2.bridge/rtx_lidar.py`。
- **RTX Radar**：**ROS 2 RTX Radar Helper** 发 PointCloud2（`radar_point_cloud`），可选径向速度/强度/时间戳元数据（需 auxiliary output level BASIC）；已知问题：Radar 忽略 `omni:sensor:tickRate` 自触发。

来源: tutorial_ros2_camera.html、tutorial_ros2_camera_publishing.html、tutorial_ros2_compressed_image.html、tutorial_ros2_rtx_lidar.html、tutorial_ros2_rtx_radar.html

## TF 与里程计

- **Isaac Compute Transform Tree → ROS 2 Publish Transform Tree**（`/tf`）；**Isaac Compute Odometry → ROS 2 Publish Odometry**（`/odom`，含 3D 速度）；**ROS 2 Publish Raw Transform Tree** 发单个/静态变换。`targetPrims` 可给单个 prim 或 articulation root（自动发整条链）；`parentPrim` 默认 /World。调试：`ros2 run tf2_tools view_frames`，或内置 TF Viewer 扩展 `isaacsim.ros2.tf_viewer`。

来源: ros2_tutorials/tutorial_ros2_tf.html

## 关节与 Ackermann

- 关节控制（Franka）：**ROS2 Publish Joint State**（→`/joint_states`）、**ROS2 Subscribe Joint State**（←`/joint_command`）、**Articulation Controller**；每个关节同一时刻只能一种控制模式（位置/速度可混用于不同关节）。测试：`ros2 run isaac_tutorials ros2_publisher.py`。
- 差速驱动（TurtleBot）：**ROS2 Subscribe Twist + Differential Controller + Articulation Controller**，订阅 `/cmd_vel`；可用 `teleop_twist_keyboard`。TurtleBot URDF 需先 xacro 预处理，导入选 Base Type=Mobile，轮子 damping 建议 1e7。
- **Ackermann**（Leatherback 车）：**ROS 2 Subscribe AckermannDrive + Ackermann Controller + Articulation Controller**，话题 `ackermann_cmd`（AckermannDriveStamped）；可用 `cmdvel_to_ackermann` 转 Twist。

来源: tutorial_ros2_manipulation.html、tutorial_ros2_drive_turtlebot.html、tutorial_ros2_ackermann_controller.html、tutorial_ros2_turtlebot.html

## 导航（含 heightmap）

- Nav2 + Nova Carter：用 Occupancy Map Generator 生成地图（Z 范围 0.1~0.62，导出 YAML+PNG）；资产 `Nova_Carter_ROS.usd` 或 `Nova_Carter_Joint_States_ROS.usd`（robot_state_publisher，扩展性更好）；编译 Isaac Sim 附带的 `carter_navigation`、`iw_hub_navigation`、`isaac_ros_navigation_goal` 包。启动：`ros2 launch carter_navigation carter_navigation.launch.py`。话题：`/tf`、`/odom`、`/map`、`/point_cloud`（经 pointcloud_to_laserscan 转 `/scan`）。
- 多机导航：靠命名空间（`/carter1~3`，OG 中设 node_namespace），`ros2 launch carter_navigation multiple_robot_carter_navigation_hospital|office.launch.py`。
- **Heightmap（6.0 新增）**：Tools > Robotics > Heightmap Importer 把 2D 占据栅格图生成带碰撞的 3D 场景，再放入 Carter + ROS_Clock 图跑 Nav2。

来源: tutorial_ros2_navigation.html、tutorial_ros2_multi_navigation.html、tutorial_ros2_navigation_heightmap.html

## FAQ 与排障

- **FAQ（6.0 新增）**：话题看不到 → 检查 ROS 发行版一致、RMW/DDS 一致、`ROS_DOMAIN_ID` 一致，且**话题只在仿真播放时发布**；发布率与物理率不符 → 用 `SimulationManager.setup_simulation`+`RenderingManager.set_dt` 统一节拍，或把 On Playback Tick 换成 **On Physics Step**，注意 `ros2 topic hz` 量的是墙钟且受 RTF 影响；仿真时间 vs 墙钟 → 发 `/clock`+`use_sim_time`；WSL2 大流量（图像/点云）延迟高，推荐原生 Linux；占据图为空 → 调 Z 范围/网格大小。来源: ros2_faq.html
- **排障**：多机导航卡顿 → 勾 Publish Full Scan 或加 `--/app/asyncRendering=true --/app/asyncPhysics=true`；深度图全黑/白 → 调 FOV 限制深度范围；发布慢 → 降渲染分辨率、开 Fabric；OmniGraph 搭好后先保存场景再仿真；Auto Namespace 对复杂层级可能不生效，用 CLI 核对。来源: troubleshooting.html
- **Isaac ROS**：docs 内该页仅是兼容矩阵（Isaac ROS 4.0.0 对应 Isaac Sim 5.x；3.2.0 对应 4.5/4.2，尚未列 6.0 条目），具体教程指向 nvidia-isaac-ros.github.io。来源: nvidia_isaac_ros/isaac_ros_tutorials.html
