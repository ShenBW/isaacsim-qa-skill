# ROS 2 集成

> 基础 URL：`https://docs.isaacsim.omniverse.nvidia.com/5.1.0/`，以下"来源"均为相对路径。

## ROS 2 Bridge 架构与支持版本

Isaac Sim 通过 `isaacsim.ros2.bridge` 扩展与 ROS 2 双向通信，官方推荐 **ROS 2 Humble 与 Jazzy**；ROS 1 支持已弃用，将在未来版本移除。支持 Python 3.11 + rclpy。四大典型场景：Nav2 导航、Python 关节控制、MoveIt 2、强化学习策略部署。

参考架构流水线：URDF 导入（开源 URDF Importer 转 USD）→ 环境搭建（上千个 SimReady 资产、Warehouse Creator）→ 传感器配置（RGB/RGBD 相机、2D/3D RTX Lidar、物理传感器）→ ROS 2 Bridge → 与外部 ROS 2 包闭环交互。两种接入方式：**OmniGraph 节点**（可视化图）与 **Python 脚本（rclpy）**。

来源：ros2_tutorials/ros2_landing_page.html、ros2_tutorials/ros2_reference_architecture.html

## 启用方式

先在系统安装并 source ROS 2，再启动 Isaac Sim 并在 Extension Manager 启用 ROS 2 Bridge；需配置 workspace 以使用自带 ROS 2 包。菜单 **Tools > Robotics > ROS 2 OmniGraphs** 可一键生成 Clock/Camera/TF/Odometry/JointStates 等常用图。

## 基于 OmniGraph 的发布/订阅机制

典型图结构：`On Playback Tick`（每帧触发）→ `ROS2 Context`（按 Domain ID 建立 DDS 通信）→ 各类 Publish/Subscribe 节点。QoS 方面，所有 ROS 2 OmniGraph 节点带 `qosProfile` JSON 字符串输入（history/depth/reliability/durability 等），也可用 **ROS2 QoS Profile** 节点选预设（Sensor Data、Default）；注意自定义 QoS 须先把 `createProfile` 设为 "Custom" 才能存入 USD。

来源：ros2_tutorials/tutorial_ros2_qos.html

## 时钟与仿真时间

用 `Isaac Read Simulation Time`（或 `Isaac Read System Time`）+ `ROS2 Publish Clock` 发布 `/clock`；外部节点设 `use_sim_time=true` 同步，如 `ros2 param set /rviz use_sim_time true`，用 `ros2 topic echo /clock` 验证。也可用 `ROS2 Subscribe Clock` 订阅外部时钟。

来源：ros2_tutorials/tutorial_ros2_clock.html

## 相机与 RTX Lidar 数据发布

- **相机（GUI）**：`Isaac Create Render Product` 获取渲染数据，`ROS2 Camera Helper` 指定数据类型与话题，可发布 RGB、深度、点云、2D/3D 包围框、语义/实例标签；`ROS2 Camera Info Helper` 发布内参（K/P/R 矩阵）。来源：ros2_tutorials/tutorial_ros2_camera.html
- **相机（编程式）**：用 `omni.replicator.core` + `ROS2PublishCameraInfo`/`ROS2PublishImage`/`ROS2PublishPointCloud` 等节点按指定频率发布（`IsaacSimulationGate` 的 step 控频，`step=int(60/频率)`）；发布 `{camera}_rgb`、`{camera}_depth`、`{camera}_camera_info`、`{camera}_pointcloud` 及 `/tf`；ROS 相机光学系与世界系用静态四元数 `[0.5,-0.5,0.5,0.5]` 连接。来源：ros2_tutorials/tutorial_ros2_camera_publishing.html
- **RTX Lidar**：菜单创建 2D/3D RTX Lidar，拖到 `base_scan` 下并清零位移；`ROS2 RTX Lidar Helper` 发布 LaserScan（`/scan`，整圈后发布）与 PointCloud2（`/point_cloud`）；RViz2 固定坐标系设 `base_scan`。注意：仿真运行中拖动 UI 窗口易崩溃，需先暂停。来源：ros2_tutorials/tutorial_ros2_rtx_lidar.html

## TF 与里程计

`ROS2 Publish Transform Tree` 发布 targetPrims 的变换；把 articulation root 加入后自动发布全部连杆，若根连杆选错需手动调整。里程计组合：`Isaac Compute Odometry Node` + `ROS2 Publish Odometry`（输出完整 3D 线/角速度）。`isaacsim.ros2.tf_viewer` 扩展可在视口内可视化 TF 树。

来源：ros2_tutorials/tutorial_ros2_tf.html

## 关节控制与机械臂 / Ackermann

- **差速驱动（TurtleBot）**：先按 URDF 教程导入 TurtleBot3（需 `sudo apt install ros-$ROS_DISTRO-xacro`；轮关节设 Velocity 目标，Damping 1e7、Stiffness 0）；再建图 `ROS2 Subscribe Twist`（订阅 `/cmd_vel`）→ `Differential Controller` → `Articulation Controller`，用 `teleop_twist_keyboard` 遥控。来源：tutorial_ros2_turtlebot.html、tutorial_ros2_drive_turtlebot.html
- **机械臂（Franka）**：`ROS2 Publish Joint State`（`/joint_states`）+ `ROS2 Subscribe Joint State`（`/joint_command`）+ `Articulation Controller`；支持位置/速度控制，每个关节同一时刻只能一种模式，靠 stiffness/damping 区分，可混合控制。来源：tutorial_ros2_manipulation.html
- **Ackermann（Leatherback）**：`ROS2 Subscribe AckermannDrive`（话题 `ackermann_cmd`，AckermannDriveStamped）→ `Ackermann Controller`（计算各轮转角与轮速；轮半径 0.052m、轮距 0.24m、轴距 0.32m、最大转角 0.7854rad）→ 两个 `Articulation Controller` 分管转向与轮速；`cmdvel_to_ackermann` 包可把 Twist 转 Ackermann。来源：tutorial_ros2_ackermann_controller.html

## 导航（Nav2 单机/多机）

- **单机**：向 Nav2 发布 `/tf`、`/odom`、`/map`、`/scan`、`/point_cloud`；用 Occupancy Map Generator 生成占据栅格（Z 上界按 Lidar 高度，Nova Carter 为 0.62m）；`ros2 launch carter_navigation carter_navigation.launch.py` 启动；RViz2 "Navigation2 Goal" 或 `isaac_ros_navigation_goal` 包设目标。Linux 全支持，Windows 部分支持。来源：tutorial_ros2_navigation.html
- **多机**：3 台 Carter 用命名空间 `carter1/2/3` 隔离（action graph 中 `node_namespace`）；launch：`multiple_robot_carter_navigation_hospital/office.launch.py`，开 3 个 RViz2 分别设 "2D Nav Goal"；CPU 占用高时启用 Lidar "Publish Full Scan" 改善同步。来源：tutorial_ros2_multi_navigation.html

## MoveIt 2

加载 Franka MoveIt 示例场景，运行 `ros2 launch isaac_moveit isaac_moveit.launch.py`（isaac_moveit 包在 Isaac Sim 自带 workspace）；RViz 中选 `panda_arm` 或 `hand` 规划组，Plan 后 Execute。已知问题：部分机器上 hand 组选 "close" 目标会执行失败；RViz 显示异常可升级 Mesa 驱动。

来源：ros2_tutorials/tutorial_ros2_moveit.html

## Standalone Python 中用 ROS 2

需先按 "Using Terminal" 设置环境变量启用 ROS 2 消息；用 `OnImpulseEvent` 节点手动触发发布以精确控频，例如 `og.Controller.set(og.Controller.attribute(".../OnImpulseEvent.state:enableImpulse"), True)`。注意每步耗时随负载变化、不匹配实时，应以仿真时钟为准。示例（`./python.sh` 运行）：Clock、相机发布、Carter Stereo、多机导航、MoveIt2、订阅示例。

来源：ros2_tutorials/tutorial_ros2_python.html

## 仿真控制服务

`isaacsim.ros2.sim_control` 扩展（启动参数 `./isaac-sim.sh --/isaac/startup/ros_sim_control_extension=True`）提供 15 个 ROS 2 服务/动作：`/set_simulation_state`（play/pause/stop）、`/step_simulation`、`/spawn_entity`、`/delete_entity`、`/load_world`、实体/状态查询等。要求 ROS 2 Humble+、Isaac Sim 5.0+、安装 `simulation_interfaces` 包。

来源：ros2_tutorials/tutorial_ros2_simulation_control.html

## Isaac ROS 生态

NVIDIA Isaac ROS 是一套 GPU 加速、高性能低延迟的 ROS 2 包集合，面向 Jetson 等平台，与 Isaac Sim 可选配合。版本兼容：**Isaac ROS 4.0.0 ↔ Isaac Sim 5.1.0/5.0.0**；3.2.0 ↔ 4.5.0/4.2.0。入口：developer.nvidia.com/isaac/ros。

来源：nvidia_isaac_ros/isaac_ros_tutorials.html

## 排障要点

- **多机导航 CPU 高/定位漂移**：Nav2 同步失败所致，启用 "Publish Full Scan" 或加 `--/app/asyncPhysics=true` 启动。
- **深度图黑白两极**：无穷远深度值拉偏对比度；RGB 失真则核对相机参数、render product 分辨率与抗锯齿。
- **MoveIt RViz 黑屏**：升级 Mesa 驱动。
- **TurtleBot 不动**：调地面/轮子摩擦系数。
- **发布频率偏低**：消息过大压垮 DDS 队列，降低 render product 分辨率。
- **RViz 收不到传感器数据**：QoS reliability 改 "Best Effort"；自定义 QoS 先设 `createProfile="Custom"`。
- OmniGraph 弃用警告可忽略；长时间运行个别场景有内存泄漏风险；运行前先保存场景。

来源：ros2_tutorials/troubleshooting.html
